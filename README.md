## AI 기술 스택

| AI 기술           | 구현 내용                       | 성능 지표                  |
| ----------------- | ------------------------------- | -------------------------- |
| RandomForest 분류 | 위험 등급 예측 (정상/주의/위험) | **5-fold CV 정확도 88.6%** |
| RandomForest 회귀 | 전국 처분인원 향후 3년 예측     | 실데이터 7년 학습          |
| Z-score 이상 탐지 | 통계적 이상 권역 자동 식별      | Z ≥ 1.5 기준               |
| 피어슨 상관분석   | 인력Risk ↔ 감염병DC 교차 분석   | 복합 위험 권역 탐지        |

> ※ 현재 ML 모델은 지방청 단위 실데이터 수가 제한적인 점을 고려하여 Risk Score 산식 기반 시뮬레이션 데이터를 증강해 검증한 MVP 모델입니다. 따라서 5-fold CV 정확도는 실제 장기 운영 데이터 기반 최종 성능이라기보다, Rule-based Risk Score와 ML 예측 간 일관성 검증 지표로 해석하는 것이 적절합니다.

## Risk Score 산출 구조

```
통합 Risk Score = 인력Risk × 40% + 감염병DC × 40% + 물자Risk × 20%
```

**인력 Risk Score**

- 2019~2025년 6년 비교 (실제 31% 감소 추세 반영)
- 행정안전부 20대 남성 비율 보정 (10% 반영)

**감염병 Disruption Coefficient (DC)**

- 지역별 감염병 발생률 지수 정규화
- 인플루엔자 유행강도 + 급성호흡기 트렌드 합산

**물자 Risk Score**

- 국외조달 의존도 + 수의계약 의존도
- 입찰참여업체 HHI 기반 공급업체 집중도

> ※ 방위사업청 조달 데이터는 지방청 단위가 아닌 전국 단위 계약·입찰 데이터이므로, 본 MVP에서는 물자 Risk를 전국 공통 외생 리스크로 산정하고 모든 지방청에 동일 적용했습니다. 지방청별 차이는 인력 Risk와 감염병 DC에서 반영하며, 물자 Risk는 병력운용 전반에 영향을 주는 공통 공급망 리스크로 통합 Risk에 20% 반영했습니다.

## 실데이터 분석 결과

최신 실데이터 분석 결과, 전체 14개 지방청 중 위험 등급은 0개, 주의 등급은 13개, 정상 등급은 1개로 산출되었습니다.

| 등급 | 개수 |
| ---- | ---- |
| 위험 | 0개  |
| 주의 | 13개 |
| 정상 | 1개  |
| 전체 | 14개 |

이는 인력 Risk는 대부분 낮은 수준이나, 감염병 Disruption Coefficient가 다수 권역에서 높게 나타나면서 통합 병력운용 Risk가 전국적으로 주의 단계로 상승했음을 의미합니다.

TriGuard AI는 개별 기관 데이터만으로는 확인하기 어려운 복합 리스크를 통합적으로 보여줍니다. 병무청 인력 데이터만 보면 대부분 정상 수준이지만, 질병관리청 감염병 데이터와 방위사업청 물자 리스크를 함께 반영하면 다수 권역이 주의 등급으로 분류되어 선제적 모니터링 필요성이 확인됩니다.

## 실행 방법

```bash
git clone https://github.com/heechan9/triguard-ai.git
cd triguard-ai
pip install -r requirements.txt
streamlit run app.py
```
